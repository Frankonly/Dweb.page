About IPFSnode

We at PACT Care set up this IPFS test node mainly for our own IOTA project called Untangle Care. But we believe that others might be interested in using IPFS for their projects as well and since there are currently only a limited number of IPFS nodes available (and almost none allows you to pin your content!), we made our node public available. However, keep in mind as soon as the demand gets too high, we might need to charge for it or shut down the service entirely.

Why IPFS + IOTA?

Storing big files permanent on the tangle is a problem since you usually end up doing a huge number of proof of works, which is quite inefficient and takes a lot of time. Instead, you just want to store a link to the file on the tangle and store the actual file with an off-tangle solution. The problem with this is that links always point to the same location, so you can not really decentralize the off-tangle solution and you will always break the previously uploaded links if you decide to switch your storage provider. That is where IPFS (InterPlanetary File System) helps. With IPFS the files don’t get a unique address which tells you where to find something, but rather they simply get a unique fingerprint called a cryptographic hash, which is similar to a unique name. This way you can simply share the unique fingerprint of a file on the tangle and can change your storage provider of the file whenever you want. Furthermore, it’s easy to decentralize the system since multiple parties can store your file with the same unique hash

Gateway

GET method:

By default, an HTTP gateway for IPFS only supports the HTTP GET method. This allows you to fetch a resource by its hash and, if the hash is a directory, by the path from that directory to a named file.

You can open files by accessing the IPFS gateway of this node under https://ipfsnode.de/ipfs/, for example https://ipfsnode.de/ipfs/QmQzCQn4puG4qu8PVysxZmscmQ5vT1ZXpqo7f58Uh9QfyY

POST method:

The Gateway of this node is a writeable gateway. So, you can send POST requests to https://ipfsnode.de/ipfs/ for example with Postman

However, keep in mind that the gateway will not pin the resulting hashes of these operations.

API (RPC)

You can access the API of this node under https://ipfsnode.de:5002/ and you can find the complete documentation of the IPFS API under the following link: https://ipfs.io/docs/api/

To integrate IPFS in your application I recommend using the appropriate client library (for example net-ipfs-api for c#).

At the moment, only the /api/v0/add endpoint is publicly available for security reasons. The great thing about it is that this endpoint actually allows you to store your content permanent on the node, so you don’t have to be scared that it’s gone after a couple of hours. If you need access to other parts of the API, feel free to reach out to me (david[dot]hawig[at]pact[dot]care).
